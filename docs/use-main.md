# Daquery User Guide

### Responding to Query Requests
Data stewards can approve queries by navigating to the Incoming Requests page, available on the left menu.

Queries are listed with their status on this page.  More details of the query can be show by clicking on the summary line for any query.

Any query listed with a yellow "PENDING" label await approval by a site data steward. A data steward can approve or deny a query by clicking the summary of the query to display the details and clicking the "Approve" or "Deny" buttons.

### Running queries
To query any node a SQL query can be submitted via the the [Outgoing Requests page](outgoing-queries.html), available on the left menu.

### Joining a network
A site can join a network by navigating to the Neworks page and clicking the "Join New Network" button.  More information is available on the [Join Network page](join-network.html).

### Managing local users
Local users can be created and edited on the [Local User page](edit-user.html).

### Managing remote users
Permission for users from connected remote sites to run aggregate queries and submit requests for data can be managed on the [Remote Users page](remote-user-roles.html).

### Installing and Updating Daquery
Daquery can be installed or updated by following the [installation instructions](install.html) or the [upgrade instructions](upgrade.html).
